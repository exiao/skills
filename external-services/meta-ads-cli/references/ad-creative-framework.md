# Ad Creative Framework and Iteration Log

Use this reference to generate and iterate paid social creative from performance data.

## Angle Categories

Before writing individual headlines or hooks, establish 3 to 5 distinct angles. Each should tap into a different motivation:

| Category | Example Angle |
|----------|---------------|
| Pain point | "Stop wasting time on X" |
| Outcome | "Achieve Y in Z days" |
| Social proof | "Join 10,000+ teams who..." |
| Curiosity | "The X secret top companies use" |
| Comparison | "Unlike X, we do Y" |
| Urgency | "Limited time: get X free" |
| Identity | "Built for [specific role/type]" |
| Contrarian | "Why [common practice] does not work" |

For each angle, vary word choice, specificity, tone, and structure.

## Iterating from Performance Data

### 1. Analyze winners

Look at top performers by the campaign's true goal, not just CTR.

Ask:
- What themes or pain points appear in top performers?
- What structures work: questions, statements, commands, numbers?
- What words or phrases recur?
- Are top performers shorter or longer?
- Which visuals stop the scroll fastest?

### 2. Analyze losers

Ask:
- What angles are not resonating?
- Are they too generic, too polished, too complicated, or too detached from the landing page?
- Did they fail at the hook, click, or conversion step?

### 3. Generate new variations

- Double down on winning themes with fresh phrasing.
- Extend winning angles into new formats.
- Test 1 to 2 new angles not yet explored.
- Avoid patterns found in underperformers.

### 4. Document the iteration

```md
## Iteration Log
- Round: [number]
- Date: [date]
- Top performers: [list with metrics]
- Winning patterns: [summary]
- New variations: [count] headlines, [count] hooks
- New angles being tested: [list]
- Angles retired: [list]
```

## Writing Quality Standards

Strong headlines and hooks are:
- Specific: "Cut reporting time 75%" beats "Save time"
- Benefit-led: "Ship code faster" beats "CI/CD pipeline"
- Active: "Automate your reports" beats "Reports are automated"
- Concrete: use numbers, timeframes, or clear before/after states when true

Avoid:
- Jargon the audience will not recognize
- Claims without specificity
- All caps or excessive punctuation
- Clickbait the landing page cannot deliver on
- Personal-attribute targeting language prohibited by Meta policies

Descriptions should complement headlines:
- Add proof points
- Handle objections
- Reinforce the CTA
- Add urgency only when genuine

## Batch Generation Workflow

For large creative batches:

1. Pull recent performance data.
2. Build an exclusion list of used hook + format + concept combinations.
3. Generate concepts across multiple angles.
4. Create assets or prompts.
5. QA for readability and policy risk.
6. Save a manifest.

Manifest fields:
- File name
- Angle
- Format
- Hook
- Primary text
- Headline
- Description
- CTA
- Audience segment
- Product promise
- Compliance notes
- Source tool
- QA status

## Native Formats

- Notes app screenshot
- Text over video
- Comment reply
- Reddit-style post
- Tweet/X screenshot style
- Meme comparison
- Testimonial card
- Product proof reveal
- Before/after workflow
- Founder or customer talking head
- Data card

## QA Checklist

Before upload:
- Text is readable on mobile
- One idea per asset
- Landing page matches the promise
- No unsupported claims
- No fake partnership or platform endorsement
- Required disclosures are present
- Visual does not look like generic AI slop
